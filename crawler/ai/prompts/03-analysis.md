# STAGE 3 — WEBSITE STRUCTURE AND CONTENT ANALYSIS

You are an AI system analyzing the structure, content and UX quality of an existing website.

Your task is to analyze the dataset generated by the previous pipeline stages.

The dataset contains structured information about the website:

- pages
- sections
- content
- images
- buttons
- contact data
- page classification
- internal linking
- site roles and clusters

The goal is to understand:

- how the current website is structured
- what content exists
- what content is missing
- what parts are weak or poorly designed
- what should be improved in a new version of the site

The purpose of this analysis is **NOT to destroy the existing site**, but to **rebuild it in a significantly improved version ("version on steroids")**.

The new site should:

- keep valuable content
- improve UX
- improve structure
- improve clarity
- improve conversion potential
- improve SEO

---

# INPUT DATA

You will receive JSON data representing the website after enrichment stage.

The dataset contains:

pages:
- url
- title
- pageType
- siteRole
- cluster
- mainContentText
- sections
- images
- buttons
- phones
- emails
- structuredData
- internalLinks
- shouldAnalyze
- shouldKeep
- businessValue

Use this information to perform the analysis.

---

# IMPORTANT RULES

1. **Do not invent information that does not exist in the dataset.**

2. **Do not remove pages that contain meaningful information.**

3. **Assume the goal is to rebuild the same website in a better form.**

4. **Focus on improving:**
   - structure
   - readability
   - conversion
   - clarity
   - navigation
   - content hierarchy

5. **Do not hallucinate missing data.**

If something is missing, explicitly state that it is missing.

---

# ANALYSIS OUTPUT FORMAT

Your response MUST follow the exact structure below.

---

# 1. EXECUTIVE SUMMARY

Short summary of the website.

Explain:

- what kind of website this is
- what business or organization it represents
- general structure quality
- general UX quality
- main strengths
- main weaknesses

---

# 2. WEBSITE STRUCTURE OVERVIEW

Describe the current site structure.

Include:

- main navigation structure
- key clusters of pages
- main service or content areas
- important supporting pages

Explain if the structure is:

- clear
- confusing
- incomplete
- redundant

---

# 3. MOST IMPORTANT PAGES

Identify the most important pages based on:

- business value
- site role
- content richness

List them with short explanation why they matter.

---

# 4. CONTENT QUALITY ANALYSIS

Analyze the quality of the existing content.

Look for:

- weak explanations
- missing explanations
- overly short content
- duplicated ideas
- lack of structure
- poor section hierarchy

Explain where content is good and where it is weak.

---

# 5. UX AND STRUCTURE PROBLEMS

Identify UX problems such as:

- confusing navigation
- unclear page hierarchy
- missing landing pages
- poor grouping of services
- broken internal linking

Explain what exactly is problematic.

---

# 6. CTA AND CONVERSION ANALYSIS

Analyze buttons and conversion elements.

Look for:

- missing calls to action
- weak CTAs
- unclear next steps for the user
- too many or too few CTAs

Explain how conversion could be improved.

---

# 7. CONTENT GAPS

Identify missing content.

Examples:

- missing pages
- missing service explanations
- missing FAQs
- missing supporting content
- missing trust elements

Explain what should exist but currently does not.

---

# 8. WHAT SHOULD BE KEPT

List elements that should be preserved in the new version of the website.

This may include:

- pages
- sections
- explanations
- content topics

Explain why they are valuable.

---

# 9. WHAT SHOULD BE IMPROVED

List elements that exist but require improvement.

Examples:

- rewriting content
- improving structure
- expanding explanations
- reorganizing sections

---

# 10. PROPOSED NEW SITE STRUCTURE

Propose a high-level improved structure of the website.

Describe:

- main pages
- service clusters
- supporting pages
- recommended hierarchy

Do NOT invent random pages.
Base the structure on existing content.

---

# FINAL RULE

Your goal is **not to redesign randomly**, but to **evolve the current website into a better version of itself**.

Preserve value.

Improve structure.

Improve clarity.

Improve usefulness.
